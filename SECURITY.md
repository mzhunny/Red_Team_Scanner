# Security Policy

## Authorized Use Only

RedTeam Sentinel is intended for defensive security learning, portfolio demonstration, and authorized testing only.

Do not use this project, its documentation, or its concepts to scan, probe, enumerate, or assess systems unless you own them or have explicit written permission.

## What This Project Is Designed For

- Security QA portfolio demonstration
- Safe scanning workflow design
- IAM/GRC control documentation
- Agentic AI security governance examples
- Compliance mapping practice
- Audit evidence planning

## What This Project Is Not Designed For

- Unauthorized scanning
- Exploitation
- Credential attacks
- Brute force testing
- Malware behavior
- Destructive testing
- Real-world red team operations without approval

## Safe Testing Boundaries

Any testing should be limited to:

- Localhost
- Lab environments
- Intentionally vulnerable training apps you control
- Systems where you have explicit written authorization

## AI Safety Boundary

AI-assisted findings must be reviewed by a human before any access, security, compliance, or remediation decision is made.

The AI agent must not:

- Approve access
- Remove access
- Accept risk
- Submit audit evidence
- Claim unsupported evidence exists
- Execute unauthorized actions

## Reporting Security Issues

If this repository is extended into runnable code and you discover a security issue, open a GitHub issue with:

- Description of the issue
- Steps to reproduce in a safe lab environment
- Expected behavior
- Actual behavior
- Recommended fix

Do not include secrets, tokens, passwords, or real sensitive data in any report.
